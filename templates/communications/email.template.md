# Email template

> Pick the variant, delete the rest. Lead with the ask; respect the reader's time.

**Subject:** `[<AREA>] <outcome-oriented summary>`  ·  e.g. `[Payments] Decision needed: gateway vendor by Fri`

**Audience:** <who> | **Type:** Status update · Decision request · Heads-up · Incident comms

---

**TL;DR** (1–2 sentences — the whole point, readable on a phone notification.)

**Context** — what changed / why you're writing, with links to the source of truth.

**The ask** — exactly what you need and by when. If a decision: state the options
and your recommendation.

**Details** — only what's needed to act; push depth to linked docs.

**Action items**

| Owner | Action | Due |
| :--- | :--- | :--- |
| @name | <action> | YYYY-MM-DD |

---

### Variant: Status update
- **Shipped / progress:** …
- **Next:** …
- **Blockers / risks:** … (and the help you need)

### Variant: Incident comms
- **Impact:** <who/what, since when> · **Severity:** SEV-n
- **Status:** Investigating | Mitigated | Resolved
- **Current action / ETA:** …
- **Next update by:** <time>
